回答にあたっての注意

  • サーバサイドはMacOSもしくはLinuxで動作するようにしてください。
  • クライアントサイドはHTMLベースで、Google ChromeもしくはFirefoxで動作するようにしてください。
  • 課題1-3はインターネット上の他のウェブサービスに依存せずに動作するようにしてください。
  • 課題1-3はライブラリやデータベースなどを自由に使用して頂いて構いません。ただし、それらの導入手順は提出するドキュメントで示してください。
  • プログラムの回答には以下のいずれかの言語を利用してください。
    • C, C++, Python, PHP, Ruby, Go, Java, JavaScript (AltJSを含む)
    • 複数の言語を使用しても構いません。
  • 課題には自分だけで取り組んでください。この課題を他の応募者を含めた他人と共有・相談することを禁止します。漏洩の証拠が見つかった場合、その応募者は失格となります。ある応募者が別の応募者に回答をコピーさせた場合、双方の応募者が失格となります。
  • 想定所要時間は2日です。全課題が解けていなくても提出は可能ですので、学業に支障の無い範囲で取り組んで下さい。

提出物

以下のものを提出して下さい。

  • 課題1-3の制作物
    • 制作物はサーバサイド・クライアントサイドのソースコード以外に、データベースの設定ファイルや事前処理に使ったプログラムのソースコードなども含まれます。
    • ライブラリは、導入方法をドキュメントに示せば、含めなくて構いません。
    • 課題で使う動画データは含めないでください。
  • 制作物のビルドに必要な環境・実行手順について記したテキスト

評価基準

提出物は以下を満たしていることが望ましいです。(必須では無いので、時間がない場合はこれらを満たしていなくても大丈夫です)

  • プログラムを実行して、課題の要求が達成できること。
  • プログラムが正しいことを検証する方法が示されていること。
  • 他人が提出物を見て、追試がしやすい形になっていること。
  • プログラムが他人が読んでも理解しやすいものになっていること。
  • ドキュメントがわかりやすく記述されていること。

また、面接当日はデモやコードレビューを実施します。どちらも行える環境を準備してください。

提出方法

問題文

本課題では、セミナー発表のような動画に対して、発言内容のアノテーション（メタデータ作成）を行うウェブサービスのプロトタイプを作成してもらいます。このプロトタイプでは、外部のクラウドサービスを使わないでください。

指定される要件が満たされていれば、どのようなUIでも構いません。定義されていない挙動は、UXが向上するように実装を行ってください。

課題1（必須）

動画ファイルをサーバサイドにアップロードし、閲覧、削除できるウェブサービスのプロトタイプを作ってください。

画面イメージ

要件が満たされていれば、画面が一致する必要はありません。

要件

  • プロトタイピングであるため、デモの範囲のアクセス数で動作する実装で構いません
  • 少なくともMP4ファイルの登録、閲覧、削除ができること
    • サービス内部の格納形式は問いません

ドラッグアンドドロップによるアップロード、アップロードの進捗表示など、UXの向上は加点項目なので、自由に行ってください。

課題2（必須）

課題1で作ったウェブサービスに、アップロードした動画の発話のアノテーションを行う機能を追加してください。

画面イメージ

要件が満たされていれば、画面が一致する必要はありません。

要件

  • アノテーションは、発話内容以外に、開始時間、終了時間、種類（通常の発話、有声休止（「えーと」「あのー」など）、ノイズ（拍手、タイムキーパーのベルなど））がつけられること
  • アノテーションは編集されるとサーバサイドに保存されること
  • これらのアノテーションが後述するJSON形式でダウンロードできること 
    [
  {
    "start_time": 1.11, // 動画における開始秒数をfloatで取る
    "finish_time": 1.23, // 動画における終了秒数をfloatで取る
    "category": "pause", // speech, pause, noise のいずれかの値を取る
    "content": "えー", // 発話内容
  },
  {
    "start_time": 1.23,
    "finish_time": 3.98,
    "category": "speech",
    "content": "それでは発表をはじめます",
  },
  {
    "start_time": 3.87,
    "finish_time": 4.32,
    "category": "noise",
    "content": "（拍手）",
  }
    : // 以後繰り返す
]

課題1の動画アップロードからSPAで構成するなど、UXの向上は加点項目なので、自由に行ってください。

課題3（自由課題）

課題2までで作ったアノテーション機能のUXがまだ不十分だと仮定します。

例えば、

  • 動画をコマ送りや早送りして作業しやすくする
  • 音声波形を表示して、発話の開始終了時点をわかりやすくする
  • 複数人での作業で、他の人が作業している区間をわかりやすくする

などの改善が考えられます。

ここに書かれた改善に限らず、他の改善を行っても構いません。UXが改善するような実装を少なくとも一つ行い、ドキュメント中でアピールをしてください。
