## テーマ別課題
ブラウザで動作する圧縮アーカイブされた画像データセットの管理ウェブサービスをPoC実装してください。

### 必須仕様

* ローカルPCにある任意のPNG, JPEGを含むZIPファイルをアップロードすると、サービス側にファイルが登録されること
* サービスに登録されたファイル名をクリックすると、サービス側で展開した画像一覧が閲覧できること
* サービスの画像一覧の任意の画像をクリックすると、オリジナルサイズの画像が確認できること
* 形式上サポートしているが、実装上動作しないファイルがアップロードされた場合はUI上で通知すること
* サービスはリポジトリのルートディレクトリで `docker compose up` を実行すると、http://localhost:12345/ でアクセスできること
    * サーバーサイドはこだわりがない場合はPythonで実装するのが望ましいです
* クライアントサイドはTypeScript+Reactで実装し、PCのChromeもしくはFirefoxで動作すること
    * これ以外のライブラリやデザインフレームワークは自由です

これ以外の構成や挙動に指定はありません。例えば、

* アップロードやオリジナルサイズの画像確認のUIに制限はありません。ドラッグ&ドロップやモーダルなど利用し実装して構いません
* 対応ファイルを増やすことに制限はありません。データセットファイルを処理するのに適当な規定を作ってください
* 画像一覧の表示方法に指定はありません。タイル状に並べられて快適に動作するように実装してください
* エラー処理に指定はありません。未対応ファイルが上げられた場合などは適切に処理してください

このような未定義な挙動のうち、ツールにおいて重要なものはREADMEに記載してください。

本課題は実現された機能以外ではコードの可読性や保守性を重視します。アプリケーションのデザインやデータセット閲覧に関わらない機能の実装はあまり大きく評価しません。最小限でうまく動作するビューワサービスを実装してください。

### テストケース

2003年に作成された[Caltech 101](https://data.caltech.edu/records/mzrjq-6wc02)データセットのうち1カテゴリが動作することが求められます。
例えば、Leopardsカテゴリを圧縮したLeopards.zipは以下の手順で生成でき、このファイルが閲覧できる必要があります。
```sh
# caltech-101.zipをダウンロードしたディレクトリに移動
unzip caltech-101.zip
cd caltech-101
tar -xzvf 101_ObjectCategories.tar.gz
cd 101_ObjectCategories
zip -r Leopards.zip Leopards
mv Leopards.zip
cd ..
```

もちろん、 `zip -r 101_ObjectCategories.zip 101_ObjectCategories` で生成したzipファイルや101_ObjectCategories.tar.gzそのもの、二重アーカイブされているcaltech-101.zip自体が処理できるように実装することは歓迎です。
