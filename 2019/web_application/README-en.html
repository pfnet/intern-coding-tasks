Task description

In this task, you are asked to develop a prototype of a web service that produces data for machine learning (annotation). This prototype has a UI for workers to create annotations in specific areas of an image and tools for data management excluding authentication. Do not use any external cloud services.

The application can have any kind of UI as long as it satisfies the given requirements. Outside of these requirements, you are free to make your own design choices but you should try to maximize the user experience (UX).

Problem 1

Develop a web UI that allows users to create square-based annotations for a given image. The image is divided into a given number of square tiles and the interface should allow annotators to specify if a particular object is contained within a tile or not.

How to input the image and return the results is left to your discretion, but bear in mind that your implementation will be used for problem 2.

Example for reference

Below is an example image from a PFN tweet, divided into 8 rows of tiles. Annotations are created by adding squares in which a “Person” is included.

It is not necessary to reproduce the same UI as above, as long as the requirements are satisfied.

Requirements

  • Display the given item label (e.g. "Person").

  • Divide the given image into the specified number of vertical divisions.

    • The image size is not fixed, so no number for horizontal divisions is provided.

  • Click a square to toggle the annotation status (item included or not in the square).

  • Properly deal with cases when the aspect ratio of the image does not allow perfect divisions into squares horizontally.

You are also encouraged to add improvements for better UX.

Problem 2

Based on the UI you created for Problem 1, develop a web service to create and distribute an annotation task among a limited number of specific annotators.

Since this is a prototype, you are not required to implement an authentication scheme. You may also assume that the number of annotators is sufficiently low so that no traffic bottlenecks occur.

How to store the image and results is left to your discretion, but bear in mind that your implementation will be used for problem 3.

Requirements

  • Create an annotation task by uploading multiple image files and inputting the number of vertical divisions as well as the annotation label.
  • An annotator can go to a URL associated with a specific annotation task to resume annotating using the UI of Problem 1.
  • Properly deal with corner cases, such as trying to upload an invalid image, assigning an image upon simultaneous access, two annotators annotating the same image.

You are also encouraged to add improvements for better UX.

Problem 3

Implement a tool to manage the results of the annotation tasks as an addition to the web service you created for Problem 2.

Requirements

  • Possibility to check the progress of an annotation task and its results in a web browser even while the task is ongoing.

  • The annotation data can be downloaded in the following JSON format:

    xxxxxxxxxx
    {
      "task": "Person", // task name
      "divide": 5, // number of vertical divisions
      "images": [
        {
          "pathname": "photo001.jpg", // annotated image file name
          "regions": [
            // The clicked area described by its upper left corner (x, y) and dimensions (w, h)
            {
              "x": 100,
              "y": 100,
              "w": 50,
              "h": 50
            },
              : // repeat
          ]
        },
          : // repeat
      ]
    }

You are also encouraged to add improvements for better UX.

Problem 4 (Optional)

There are many ways this annotation web service can be further improved. Such as:

  • Remove low quality annotations on a per-user basis. Implement a mechanism for administrators to remove annotations from a specific user using tracking methods such as cookies etc.
  • Support distributed annotation of very large images by dividing such images into multiple pieces/tasks, having them annotated by separate workers and finally merging the annotations into a single data file.
  • Possibility for administrators to monitor how annotators perform their annotation tasks. Implement action tracking in the annotator UI and a feature to view the actions in some way in the administrator UI.

The above enhancements are only suggestions and you may make your own improvements to the service. Please implement at least one improvement for better UX, and explain its benefits in your documentation.
